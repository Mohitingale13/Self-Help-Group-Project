You are a senior full-stack Flutter developer.

Add a feature to the existing SHG Digital Platform that allows downloading a structured PDF statement for each member.

IMPORTANT:
The PDF must be professionally formatted, structured, and audit-ready.
It must not be a simple raw dump of data.
It should look like an official SHG record statement.

========================================================
FEATURE NAME:
"Download Member Statement"
========================================================

WHERE IT SHOULD APPEAR:
Inside Member Profile Page:
Add a button:
[ Download Full Statement (PDF) ]

========================================================
PDF CONTENT STRUCTURE
========================================================

The PDF must contain the following structured sections:

--------------------------------------------------------
1. HEADER SECTION
--------------------------------------------------------
- SHG Group Name
- Unique Group ID
- Village Name
- President Name
- Generated Date
- System Name: SHG Digital Record Platform

--------------------------------------------------------
2. MEMBER DETAILS SECTION
--------------------------------------------------------
- Member Name
- Phone Number
- Village
- Join Date
- Planned Exit Date
- Current Status (Active / Left)
- Total Savings Paid
- Total Loan Taken
- Outstanding Loan Amount

--------------------------------------------------------
3. SAVINGS HISTORY TABLE
--------------------------------------------------------
Table Columns:
- Date
- Amount
- Status (Pending / Confirmed)
- Verified By

Chronological order (oldest to newest)

Show Total Savings at bottom.

--------------------------------------------------------
4. LOAN HISTORY SECTION
--------------------------------------------------------

For each loan:

Loan Summary:
- Loan ID
- Resolution Number
- Loan Amount
- Interest Rate
- Loan Duration
- Approved Date
- Remaining Balance

Repayment Table:
- Installment Date
- Amount Paid
- Remaining Balance

--------------------------------------------------------
5. ATTENDANCE RECORD SECTION
--------------------------------------------------------

Table:
- Meeting Date
- Attendance Status (Present / Absent)
- Notes (if any)

Show:
Total Meetings
Total Present
Attendance Percentage

--------------------------------------------------------
6. FINE HISTORY SECTION (if applicable)
--------------------------------------------------------

- Fine Date
- Reason
- Amount
- Paid Status

--------------------------------------------------------
7. FOOTER
--------------------------------------------------------

- Digitally generated record
- No manual signature required
- Generated by SHG Digital Platform
- Page numbers

========================================================
TECHNICAL IMPLEMENTATION REQUIREMENTS
========================================================

1. Use Flutter PDF package (pdf and printing packages).
2. Generate PDF locally in app.
3. Fetch all member-related data using groupId + memberId.
4. Ensure proper data sorting by date.
5. Use clean layout with:
   - Bold section titles
   - Proper spacing
   - Table borders
   - Consistent formatting
6. Format currency as ₹ (Indian Rupees).
7. Date format: DD-MM-YYYY.

========================================================
DATABASE QUERIES
========================================================

When generating PDF, fetch:

- Member document
- Payments collection filtered by memberId
- Loans collection filtered by memberId
- Loan repayments
- Attendance records
- Fine records
- Group details (for header)

========================================================
SECURITY REQUIREMENTS
========================================================

- Member can download only her own statement.
- President can download statement of any member in her group.
- Admin can download any group member statement.
- Enforce using Firebase security rules.

========================================================
UI BEHAVIOR
========================================================

When user clicks download:
- Show loading indicator.
- Generate PDF.
- Provide options:
   - Save to device
   - Share
   - Print

========================================================
OUTPUT REQUIRED
========================================================

1. Full Flutter code for PDF generation.
2. Example PDF layout implementation.
3. Firestore query logic.
4. Role-based access validation.
5. Example structured data model.

The PDF must look professional and suitable for SHG audit, NGO reporting, and record keeping.

Architecture must remain modular and scalable.
